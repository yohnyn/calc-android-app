# AI_HANDOFF

## 交接摘要

本仓库是 Android/Kotlin/Jetpack Compose 项目，产品当前为“仓位助手（Futures Calculator）优化 3.0 P1 体验增强”口径。

核心原则：

- App 是本地计算器 / 模拟器 / 对比工具，不是交易顾问。
- 不登录交易所，不绑定 API，不下单，不读取账户。
- 只展示客观计算结果，不展示投资建议、推荐方案、最佳方案、最优方案、风险等级或风险高低评价。
- 本轮 3.0 P1 只做结果详情、方案库、历史、复制、设置和低权重入口体验增强，不新增交易、行情、账户或投资建议能力。

## 关键约束

- 必须遵守根目录 `AGENTS.md`：禁止运行任何构建、编译、安装、测试命令，包括 Gradle。
- 用户明确要求：不要在线编译，不要本地编译，不提交 git。
- 允许源码阅读、局部编辑、静态推理和非编译检查。
- 完成源码任务后要同步项目文档。

- `android-app/app/src/main/java/com/personal/futurescalculator/ui/CalculatorScreen.kt`：主屏编排，接入目标价格、方案库、开仓方案对比、补仓助手等模块。

## 最近完成内容

- 优化 3.0 首页一级模块规则已同步：`仓位参数 → 结果缩略 → 开仓方案对比 → 补仓助手 → 目标价格轻入口 → 历史 / 方案库 / 设置等低权重入口`。
- 优化 3.0 P0 已确立：凡是属于当前开仓方案的字段，必须收进仓位参数。
- 首页不再单独渲染 `CoinMarketHeader` 或独立币种选择卡；币种 / 交易对已移动到 `PositionInputSection` 内部靠前位置。
- `U 本位 / 币本位` 已收进仓位参数内部轻量结算模式行。
- 首页 `结果缩略` 已从仓位参数卡片内部拆出，紧跟仓位参数，只展示净盈亏 / ROI / 查看详情。
- `HomeModule.defaultOrder` 已同步为 3.0 顺序，避免后续误用旧排序。
- 仓位参数顶部已改为左侧币种轻量入口、右侧结算模式下拉；币种区域不再突出当前价格或刷新主视觉。
- 币本位计算方式已改为首次弹窗确认并作为全局设置，设置页可修改并显示当前值；对比方案编辑不再显示该字段。
- 输入方式已合并到第一个输入项标题，开仓价与平仓价已合并为 `价格` 输入组。
- 历史页新增 `历史记录 / 方案库` 双标签。
- 方案库和历史记录分离：历史记录是自动计算快照，方案库是用户主动保存的开仓方案。
- 方案库支持打开方案、加入方案对比、删除方案。
- 方案库支持重命名、纯文本备注和复制为新方案；复制生成新 id，备注不参与计算。
- 历史详情第一屏展示结果摘要，完整参数默认折叠；收益测算历史可手动保存为方案。

- 复制战绩支持简洁版、详细版、对比摘要和补仓结果；复制默认格式支持每次询问。
- 设置页集中保留主题、盈亏配色、复制默认格式、币本位计算方式、用户反馈、关于、隐私政策、免责声明和支持作者入口。

- 方案对比添加方案入口拆成 `手动创建 / 从方案库选择`。

- 方案对比结果顶部改为 `相对主方案差值列表`，逐项展示所有选中方案相对主方案的差值。

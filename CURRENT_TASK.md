# CURRENT_TASK

## 当前任务

优化 3.0 P0 最终整合修正：首页结构与仓位参数 UI 收口；按用户截图反馈继续美化首页。

本轮是在既有 3.0 首页收口基础上，继续修正顶部币种/结算选择、仓位输入单位、全仓逐仓恢复、杠杆单一下拉、目标价格展开位置、全项目下拉菜单、价格箭头、轻量勾选反馈、支持作者入口和补仓助手入口视觉；不新增交易所 API、账户读取、下单、行情订阅或投资建议口径。

## 本轮完成项

- 仓位参数顶部币种显示仅保留图标 + 缩写，去掉全称显示。
- 币种选择和结算模式改为统一轻量选择卡，下拉箭头右侧居中。
- 做多/做空与全仓/逐仓选择已恢复到仓位参数内。
- 仓位输入改为一体式控件：左侧数字输入，右侧单位选择 `USDT / 当前币种`，中间用浅竖线分隔；取消孤立的输入模式下拉 chip。
- 杠杆只保留单一轻量下拉，不再展示独立杠杆数值卡或一整排快捷 chip。
- 所有下拉控件统一为绘制型居中 chevron，替换文字 `⌄/▼`，并修正仓位/杠杆触发区的对齐。
- `目标价格` 后增加 `强平价格` 选项，同区展示；展开内容贴在该选择行下方，不再作为页面独立模块。
- 所有直接使用的下拉菜单统一为白底、细边框、轻阴影和紧凑行距，避免默认菜单和触发控件割裂。
- 杠杆、仓位单位和结算方式下拉弹层与触发框等宽，避免展开时尺寸跳变；添加对比方案顶部复用首页币种/结算选择样式。
- `强平价格` 与 `可开` 共用账户总资金模块；开启任一功能时展示，共享值不得因关闭其中一个功能而清空；`可开` 根据杠杆、开仓价和总资金实时计算可开仓位与数量。
- 首页结果缩略标题与结果保持在同一卡片中；所有计算结果展示为整数。
- 方案库增加二次确认的 `全部删除`，并同步清理首页对比方案与补仓助手中由方案库带入的数据。
- 补仓助手的手动选择来源只允许使用方案库，不得混入历史记录、当前主方案或临时对比方案。
- 目标价格展开区改为同一行 `目标收益 USDT / 目标亏损 USDT` 两个输入框，删除填入止盈价和填入止损价功能。
- 币种图标使用不可压缩的正方形容器、圆形裁切和居中渲染，避免父布局约束导致图标扁斜。
- 首页缩略结果直接以 `净盈利 / 净亏损 + 金额 + ROI` 展示，不再使用“结果缩略”标题。
- 主结果详情第一屏移除手续费、交易费用和未扣手续费盈亏；强平价格与距离强平在仓位安全边界中同排左右展示；第二栏为交易参数，设置费率入口位于底部费率展示控件内部右侧；第三栏为计算公式；删除独立费用明细栏。
- 首页净盈利 ROI 必须显示 `+`；交易参数默认展开，设置费率入口位于底部费率展示控件内部右侧；计算公式下方展示本次实际净盈亏与 ROI。
- 用户界面将 ROI 改称 `保证金收益率`，详情使用 `保证金收益率（ROI）`；勾选强平价格后以独立 `仓位安全边界` 警示区展示在详情首屏，可计算时显示价格，不可计算时显示原因。
- 历史记录仅在用户主动打开有效结果详情时保存；输入变化和实时计算不得生成记录。
- 首页结果卡显示 `净盈利 / 净亏损`，数值格式为 `金额 · 收益率`，不重复显示收益率名称。
- 计算公式区直接展示当前这一单的净盈亏与保证金收益率计算过程，不再单独展示“本次结果”卡片；中间计算项不写 `USDT`，仅最终结果附加单位，负数中间项使用括号包裹。
- 勾选强平价格后详情始终展示仓位安全边界；无法得出正数强平价格时显示明确原因。
- 开仓价 / 平仓价中间箭头已下移居中。
- 止盈/止损、强平价格和目标价格点击时不再触发整块背景突兀变色。
- 首页底部恢复支持作者入口；历史 / 方案库 / 设置仍作为底部低权重入口，`重置` 不回到首页底部。
- 补仓助手入口恢复轻量白底卡，右侧 `展开` 文案和箭头居中。
- 补仓助手在没有主方案结果时，也可从方案库选择已保存方案填入当前持仓摘要。
- 方案库入口可直接从底部进入历史页的方案库标签。
- 继续保留币本位全局模式与添加 / 编辑对比方案不显示币本位计算方式的规则。

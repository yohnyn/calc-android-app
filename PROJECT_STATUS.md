# PROJECT_STATUS

## 项目状态

仓位助手已从基础收益计算器扩展为较完整的个人仓位模拟工具。当前处于功能收口后的本地编译、测试与真机验证阶段。

主工程：`android-app/`

## 产品边界

- 不登录交易所、不读取账户、不下单。
- 除公开币价、币种图标和用户主动打开的反馈页面外，不新增网络能力。
- 所有金融计算使用 `BigDecimal` 在本地完成。
- 只展示客观计算结果，不提供投资建议、推荐方案、最佳方案或风险等级。

## 已实现功能

### 核心计算

- U 本位与币本位仓位计算。
- 做多/做空、全仓/逐仓、杠杆、保证金或币数量输入。
- 仓位价值、所需保证金、未扣手续费盈亏、费用、净盈亏和保证金收益率。
- 可选止盈/止损、目标盈利/最大亏损反推价格、强平价格估算和可开仓位估算。
- 逐仓强平估算在主计算与补仓计算中使用同一简化公式。

### 首页与结果

- 首页直接从仓位参数开始，输入变化实时更新结果卡。
- 结果卡独立提供 `保存方案` 和 `查看详情` 操作区。
- 保存当前方案按完整参数去重，重复保存会提示。
- 结果详情突出净盈亏和保证金收益率；交易参数默认展开，公式默认折叠。
- 结果详情展示计算后的当前仓位保证金和按开仓价计算的仓位币数量。
- 复制支持简洁版、详细版、对比摘要和补仓结果。

### 开仓方案对比

- 支持手动创建或从方案库选择对比方案。
- 首页当前参数不完整时仍可使用其他完整方案进行对比。
- 任意完整已选方案可设为基准；结果和复制摘要按所选基准计算。
- 方案摘要采用紧凑可展开布局；详情支持 U 本位止盈/止损结果。
- 不展示排名、最佳方案或推荐方案。

### 方案库与历史

- 方案库保存完整输入参数，支持打开、重命名、备注、复制、加入对比、带入补仓助手和删除。
- 从方案库打开方案会记录来源；删除仍关联的来源方案会同步清理首页工作区。
- 用户手动修改首页参数、币种或结算方式后解除来源关联，避免误清独立计算。
- 全部删除方案会清理对比方案和由方案库带入的补仓状态。
- 历史记录只在用户主动打开有效结果详情时保存，与方案库分离。
- 历史和方案库删除操作均有确认流程。

### 补仓助手

- 默认折叠，普通展开不会隐式带入或覆盖首页结果。
- 当前持仓支持手动创建、从方案库选择、从结果详情 `模拟补仓` 或方案库 `带入补仓助手`。
- 手动创建的当前持仓仅用于本次补仓，不写入方案库。
- 流程按 `当前持仓 → 补仓设置 → 补仓后预览 → 可选收益模拟` 排列。
- 支持按金额、币数量和当前保证金 `25% / 50% / 100%` 快捷补仓。
- 目标平仓价可选；未填写时仍计算补仓后均价、数量、保证金和仓位变化。
- 当前持仓可清空；保证金编辑值最多两位小数，币数量最多六位小数。
- 补仓数量最多显示六位小数；比例快捷金额保留两位小数。
- 补仓结果详细数据使用紧凑双列布局。

### 数据与设置

- CoinGecko 公开币种与价格缓存、自定义币种和图标缓存。
- `SharedPreferences` 保存历史、方案库和 UI 偏好。
- 设置包含主题、盈亏配色、复制格式、币本位计算方式、反馈、关于、隐私、免责声明和支持作者。
- 设置、历史、方案库和静态页面支持系统返回。

## 当前已知风险

- 最新源码尚未由用户本地完成编译、测试和真机验证；代理按规则未运行这些命令。
- 最新变更涉及多个 Compose 调用签名和跨模块交互，需要优先确认编译状态。
- 方案来源删除联动、对比基准切换、补仓页面自动定位和小额数量显示需要真机验证。
- 币本位手续费、初始保证金和保证金收益率没有确认公式，当前不得套用 U 本位口径。
- 网络失败降级、旧本地数据兼容和深色模式仍需真机验证。
- 仓库仍跟踪 `futures-calculator/android-app/` 下的早期遗留 Kotlin 文件，处理前需要确认。

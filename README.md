# calcApp

个人使用的 Android 仓位助手，用于本地模拟 U 本位和币本位合约仓位，计算收益、亏损、保证金收益率（ROI）、保证金、强平价格、目标价格，并提供开仓方案对比、方案库、历史记录和补仓助手。

App 不登录交易所、不读取账户、不下单。除公开币价、币种图标和用户主动打开的反馈页面外，计算与数据均保留在本地。

`.aider.chat.history.md` 仅为历史档案，不参与当前规则判断。

## 打开项目

用 Android Studio 打开 `android-app/`。

环境要求：

- JDK 17 或 Android Studio bundled JDK
- Android SDK API 36
- Android Studio Gradle Sync

## 当前功能

- U 本位与币本位仓位计算，支持做多/做空、全仓/逐仓、保证金或币数量输入。
- 收益、费用、保证金收益率、强平价格、止盈/止损和目标价格计算。
- 首页实时结果卡、完整结果详情、复制结果和保存方案。
- 开仓方案对比支持手动创建、从方案库选择、独立选择基准方案和复制对比摘要。
- 方案库支持保存去重、完整参数恢复、重命名、备注、复制、加入对比、带入补仓助手和删除。
- 历史记录只在用户主动打开有效结果详情时保存，与方案库分离。
- 补仓助手支持手动创建当前持仓或从方案库/结果详情明确带入，按金额、币数量或保证金比例模拟补仓。
- CoinGecko 公开币价与币种图标缓存、自定义币种、主题、盈亏配色、复制格式和币本位计算方式设置。

## 当前交互口径

- 首页顺序固定为：`仓位参数 → 结果卡 → 开仓方案对比 → 补仓助手 → 支持作者 → 历史记录 / 方案库 / 设置`。
- 当前方案字段统一放在仓位参数内；目标价格、止盈/止损、强平价格和可开均为轻量可选项。
- 首页结果卡将 `保存方案` 与 `查看详情` 分为独立点击区；相同参数方案不会重复保存。
- 结果详情突出净盈亏和保证金收益率；交易参数默认展开，计算公式默认折叠。
- 当前仓位保证金使用计算结果；仓位币数量按开仓价计算并显示三位小数。
- 当前首页参数不完整时仍可使用方案对比；至少两个完整方案被选中后才能查看对比结果。
- 删除当前打开的方案时，若首页仍与该方案关联，会同步清空其带入的首页、对比和补仓数据；用户手动修改后解除关联。
- 补仓助手默认折叠。普通展开不会自动覆盖当前持仓；只有 `模拟补仓`、`带入补仓助手` 或用户主动选择方案才会带入数据。
- 补仓当前持仓可临时手动创建，不写入方案库；保证金编辑值最多两位小数，币数量最多六位小数。
- 补仓核心计算不要求目标平仓价；目标平仓价只用于可选收益模拟。

# calcApp

个人使用的 Android 仓位助手项目，用于本地模拟 U 本位和币本位合约仓位的收益、亏损、ROI、保证金、强平价格、目标价格、补仓助手、方案库和开仓方案对比。

## 当前最重要文档

如果你要把项目交给另一个 AI 继续开发，请让它按顺序读取：

1. `AGENTS.md`：AI/代理工具必须遵守的操作限制。
2. `AI_HANDOFF.md`：快速了解接力上下文和推荐下一步。
3. `APP_DEVELOPMENT_SPEC.md`：唯一产品范围、公式和验收规范。
4. `PROJECT_STATUS.md`：唯一长期项目状态、风险和下一步。
5. `CURRENT_TASK.md`：仅记录当前任务。

其中：

- **另一个 AI 第一份应该读：`AGENTS.md`**
- **功能验收以：`APP_DEVELOPMENT_SPEC.md` 与用户最新要求共同为准**
- **`.aider.chat.history.md` 仅为历史档案，不参与当前规则判断**

## 主要文件

- `APP_DEVELOPMENT_SPEC.md`：产品结构、功能范围、公式、约束、开发步骤和验收清单。
- `PROJECT_STATUS.md`：项目长期状态，记录已完成内容、已知问题和下一步。
- `CURRENT_TASK.md`：当前/最近一次任务记录。
- `AI_HANDOFF.md`：给下一位 AI 或开发者的交接说明。
- `AGENTS.md`：AI 工作规则，尤其是禁止执行构建/编译/测试命令。
- `android-app/`：Android 主工程目录。

## 打开项目

用 Android Studio 打开：

```text
android-app
```

如 Android Studio 提示，请安装 Android 16 SDK / API 36。

## 环境需求

- JDK 17 或 Android Studio bundled JDK。
- Android SDK API 36。
- Android Studio Gradle Sync。

## 重要限制

所有代理操作限制以 `AGENTS.md` 为唯一依据。

当前仓库的 Android 项目禁止由代理运行 Gradle、编译、安装或测试命令；用户本地自行执行这些操作。

## 当前 UI 口径

- 优化 3.0 P0 已确立：当前开仓方案字段必须收进仓位参数。
- 优化 3.0 P1 已补齐体验增强：结果详情第一屏突出净盈亏与保证金收益率，强平价格和距离强平同排展示；交易参数默认展开，计算公式默认折叠，不设置独立费用明细栏；方案库支持重命名、备注和复制为新方案；历史详情支持保存为方案；复制战绩支持简洁版、详细版、对比摘要和补仓结果。
- 首页直接从仓位参数开始，不做营销式标题；币种 / 交易对属于仓位参数内部字段。
- 首页一级模块固定为：仓位参数、结果缩略、开仓方案对比、补仓助手、支持作者、历史 / 方案库 / 设置等低权重入口；目标价格由仓位参数里的 `目标价格` 小复选框展开。
- 仓位参数顶部采用左侧币种轻量入口、右侧结算模式下拉。
- 仓位输入采用数字输入 + 单位选择的一体控件，默认 USDT，可切换当前币种。
- 所有下拉触发统一使用居中 chevron 图标，不再混用文字箭头。
- 杠杆、仓位单位和结算方式下拉菜单与触发框等宽；添加对比方案顶部复用首页同款币种/结算选择。
- `强平价格` 与 `可开` 共用账户总资金；`可开` 根据杠杆、开仓价和总资金实时计算可开仓位与数量。
- 所有计算结果展示为整数，内部仍使用 `BigDecimal` 计算。
- 方案库支持二次确认的全部删除；删除全部方案时同步清理首页对比方案及补仓助手方案来源。
- 补仓助手手动选择仅读取方案库；当前主页主方案在展开时自动带入。
- 目标价格展开区直接使用同一行目标收益/目标亏损输入框，不提供填入止盈价或填入止损价按钮。
- 币本位计算方式是全局设置，首次切换时弹窗确认，之后在设置页修改。
- U 本位 / 币本位只作为仓位参数内部的轻量结算方式行，不作为独立大卡片。
- 首页结果卡直接展示净盈利/净亏损、金额与保证金收益率；结果详情第一屏不展示手续费或交易费用，强平价格与距离强平同排左右显示；第二栏交易参数默认展开，设置费率入口位于费率展示控件内部右侧，第三栏展示当前单的代入计算过程。
- 用户界面将 ROI 称为 `保证金收益率`，详情等解释性位置显示 `保证金收益率（ROI）`；主动勾选强平价格后，详情首屏使用独立 `仓位安全边界` 警示区展示，可计算时显示价格和距离，不可计算时显示明确原因。
- 历史记录仅在用户主动打开有效结果详情时保存，输入变化与实时计算不会生成记录。
- 首页结果卡使用 `净盈利/净亏损 + 金额 · 收益率`；计算公式区展示当前这一单的代入计算过程，中间项不带 `USDT`，仅最终结果附加单位，负数中间项使用括号包裹。
- 勾选强平价格后详情始终显示仓位安全边界；无法计算时显示明确原因。
- 输入变化后实时更新首页缩略结果，不依赖键盘回车、Done、失焦或收起键盘；详细弹窗只由用户主动打开。
- 保证金/币数量使用轻量 radio 输入方式切换。
- 止盈/止损、强平价格与目标价格使用小复选框文字开关，不做大按钮。
- 补仓助手从主方案带入当前仓位只读摘要，用户只补充补仓价格、补仓金额/数量和目标价。
- 历史记录与方案库分离；清空历史记录和方案库删除都必须二次确认。
- 方案对比只在至少两个有效方案被选中时展示对比结果入口。
- 补仓助手在没有主方案结果时，也可从方案库选择已保存方案填入当前持仓摘要。
- 首页底部和设置页都保留支持作者入口。
- 设置页集中放主题、盈亏配色、复制默认格式、币本位计算方式、用户反馈、关于 App、隐私政策和免责声明。
